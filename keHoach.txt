Các Giai Đoạn Phát Triển
Giai đoạn 0 — Project Foundation
Mục tiêu: Có môi trường dev chạy được, render được thứ gì đó trên Quest 3.
Việc cần làm:

Khởi tạo Vite + Three.js
Cấu hình HTTPS local (bắt buộc cho WebXR)
Tạo WebXR AR session immersive-ar cơ bản
Renderer alpha transparent — thấy phòng thật qua passthrough
Vòng lặp requestAnimationFrame XR-aware
Deploy lên Vercel, test trên Quest Browser thật

File tạo ra: main.js, core/XRSession.js, core/Renderer.js, index.html, vite.config.js
Test: Vào URL trên Quest → bấm "Enter AR" → thấy phòng thật, có 1 hình lập phương ảo nổi trong không khí → pass.

Giai đoạn 1 — Boundary & World Setup
Mục tiêu: Game lấy vùng chơi từ Guardian boundary của Quest 3, detect bàn để đặt vật thể.
Việc cần làm:

Request XRBoundedReferenceSpace (bounded-floor), fallback local-floor nếu thất bại
Visualize boundary polygon trên sàn (mesh mờ màu xanh)
Tích hợp WebXR Plane Detection API — detect table / wall (không cần floor)
Phân loại plane: wall / table
Tích hợp Mesh Detection: lấy room mesh, render wireframe debug
Hiển thị UI "đang quét phòng..." khi chưa có room mesh

Dùng lại từ GĐ0: XRSession, Renderer, vòng lặp XR
File tạo ra: world/PlaneDetector.js, world/MeshDetector.js, world/SceneManager.js
Test: Vào AR → thấy boundary Guardian hiển thị trên sàn → quét phòng → table plane được nhận diện → wireframe room mesh hiện → pass.

Giai đoạn 2 — Player Rig & Controller Input
Mục tiêu: Người chơi có tay ảo, controller điều khiển được.
Việc cần làm:

Camera theo headset pose chính xác
Render 2 controller / tay ảo đúng vị trí tay thật
Đọc toàn bộ input: thumbstick, trigger, grip, button A/B/X/Y
Map thành named action events (move, turn, grab, shoot, reload)
Snap turn bằng thumbstick phải
Teleport cơ bản bằng ray pointer — chỉ cho teleport trong vùng boundary Guardian
Comfort vignette khi xoay

Dùng lại từ GĐ1: boundsGeometry cho giới hạn teleport
File tạo ra: player/PlayerRig.js, player/ControllerInput.js, player/Locomotion.js, player/HapticManager.js
Test: Đứng trong phòng → xoay bằng thumbstick → controller render đúng vị trí tay → bấm trigger thấy log event "shoot" → bấm grip thấy "grab" → teleport đến điểm trong boundary → teleport ra ngoài boundary bị chặn → pass.

Giai đoạn 3 — Hand Tracking & Gesture
Mục tiêu: Chơi được bằng tay không, không cần controller.
Việc cần làm:

WebXR Hand API: đọc 25 khớp mỗi bàn tay
Render hand mesh theo khớp xương
Tính velocity cổ tay mỗi frame
Detect gesture cơ bản: Punch, Chop, Block, Grab (pinch), Power Slam
Emit gesture event có confidence score
Fallback mượt mà về controller khi mất tracking
Debug overlay hiện tên gesture đang detect

Dùng lại từ GĐ2: Named action event system (gesture map vào cùng event với controller)
File tạo ra: player/HandTracking.js, player/GestureDetector.js
Test: Bỏ controller → thấy tay ảo theo tay thật → đấm nhanh → log "PUNCH detected, velocity: 3.2m/s" → chụm ngón tay → log "GRAB" → giơ 2 tay lên → log "BLOCK" → pass.

Giai đoạn 4 — Physics World & Room Collision
Mục tiêu: Physics hoạt động, vật thể ảo tôn trọng phòng thật.
Việc cần làm:

Khởi tạo Rapier physics world
Feed room mesh từ MeshDetector (GĐ1) vào Rapier làm static collider
Thả quả bóng ảo từ trên cao → rơi xuống sàn thật, nảy lên, không xuyên tường
Object pool cơ bản (dùng cho đạn và particle sau này)
Debug draw collider wireframe để verify đúng với phòng thật

Dùng lại: MeshDetector room mesh · PlayerRig vị trí tay để thả bóng từ tay
File tạo ra: core/PhysicsWorld.js, utils/ObjectPool.js, utils/MathUtils.js
Test: Bấm trigger → bóng rơi từ tay → chạm sàn thật nảy lên → lăn vào góc tường thật dừng lại, không xuyên → pass.

Giai đoạn 5 — Gun System
Mục tiêu: Cầm súng thật sự, bắn ra đạn có vật lý.
Việc cần làm:

Load GLB model súng (pistol trước)
Đặt súng lên mặt bàn thật (dùng table plane từ PlaneDetector GĐ1)
Pickup: đưa tay gần → grip/pinch để cầm → súng bám theo tay
Drop: thả grip → súng rơi theo Rapier physics xuống sàn thật
Bắn: trigger → spawn đạn từ nòng → đạn bay theo Rapier rigidbody
Đạn va chạm với room mesh (tường thật) tạo impact spark
Muzzle flash particle
Ammo count, hết đạn không bắn được
Reload gesture: tháo băng đạn tay trái chạm vào súng tay phải

Dùng lại: PlaneDetector (table plane), ControllerInput, GestureDetector, PhysicsWorld
File tạo ra: combat/GunSystem.js, combat/BulletManager.js, weapons/Pistol.js
Test: Súng nằm trên bàn thật → đưa tay đến gần → grip → súng theo tay → bắn 5 phát → thấy đạn bay, spark trên tường thật → hết đạn → reload → bắn tiếp → pass.

Giai đoạn 6 — Enemy Cơ Bản & Hit Detection
Mục tiêu: Có target để bắn, bắn trúng có phản ứng.
Việc cần làm:

Enemy dummy đứng yên trên sàn thật (spawn trong boundary)
Hitbox phân vùng: đầu, thân, tay chân
BVH hit detection: đạn chạm hitbox → trừ HP
Headshot nhân 2 damage
HP bar nổi trên đầu enemy (VRUI cơ bản)
Hiệu ứng trúng đạn: flash đỏ, particle máu nhỏ
Enemy chết: ngã xuống đơn giản (chưa ragdoll)
Haptic khi bắn trúng

File tạo ra: combat/HitDetection.js, enemy/EnemyAI.js (stub), ui/VRUI.js, utils/Logger.js
Test: Enemy đứng trên sàn thật → bắn vào thân → HP giảm → bắn vào đầu → HP giảm gấp đôi → HP về 0 → enemy ngã → pass.

Giai đoạn 7 — Melee Combat
Mục tiêu: Đấm tay không được, kết hợp được với súng.
Việc cần làm:

Tính velocity tay mỗi frame (controller lẫn hand tracking)
Sphere cast quanh nắm đấm detect va chạm với enemy hitbox
Damage tỉ lệ velocity: dưới ngưỡng = không damage
Knockback enemy khi nhận đòn mạnh
Combo detector: 3 đòn trong 2 giây → combo bonus damage
Haptic tay khi đấm trúng
Tay trái bắn súng, tay phải đấm cùng lúc hoạt động độc lập

File tạo ra: combat/MeleeSystem.js
Test: Đấm chậm → không damage → đấm nhanh → HP giảm + controller rung → đấm liên tiếp 3 cái nhanh → log "COMBO x3" → tay phải đấm, tay trái bắn cùng lúc → cả 2 đều damage → pass.

Giai đoạn 8 — Enemy AI & Pathfinding
Mục tiêu: Enemy tự di chuyển, không đi xuyên tường thật.
Việc cần làm:

StateMachine: patrol -> aggro -> Stagger -> die
Patrol: di chuyển giữa các waypoint ngẫu nhiên trong boundary
Aggro: phát hiện người chơi bằng line-of-sight check (dùng room mesh)
Pathfinding đơn giản trên boundary area, né room mesh collider
Enemy tự tìm đường đến gần người chơi, di chuyển tránh các vật cản, chạm vào người chơi sẽ gây dmg cho người chơi
Stagger: enemy nhận đủ damage liên tiếp → dừng lại 1 giây
Nhiều enemy cùng lúc hoạt động độc lập

File tạo ra: enemy/StateMachine.js, enemy/EnemyAI.js (hoàn chỉnh), enemy/EnemyPool.js, enemy/EnemySpawner.js
Test: Spawn 3 enemy → patrol quanh phòng → bước vào tầm nhìn → chúng lao đến → enemy né qua ghế thật, không xuyên.

Giai đoạn 9 — Enemy Die & Audio
Mục tiêu: Enemy chết tự nhiên, âm thanh có vị trí 3D.
Việc cần làm:

Khi die: enemy rơi xuống sàn
Xác va chạm sàn thật, không xuyên
Fade out xác sau 5 giây
Tích hợp Howler.js HRTF spatial audio
Sound bank: gunshot, reload click, punch impact, enemy growl, footstep
Âm thanh enemy phát ra từ đúng vị trí 3D của chúng
Reverb ước tính theo kích thước phòng

File tạo ra: enemy/Ragdoll.js, audio/AudioManager.js, audio/SoundBank.js
Test: Bắn enemy từ bên trái → xác đổ về đúng hướng đạn → chạm sàn thật nảy nhẹ → nghe tiếng súng từ đúng phía → xác fade out sau 5 giây → pass.

Giai đoạn 10 — Wave System & Full HUD
Mục tiêu: Game có vòng lặp hoàn chỉnh, chơi được từ đầu đến cuối.
Việc cần làm:

Wave manager: đếm enemy còn sống, trigger wave mới khi = 0
Mỗi wave: tăng số lượng, tốc độ, thêm loại enemy mới
Nghỉ 10 giây giữa wave, spawn power-up trên bàn thật (dùng table plane)
Thêm 2 loại súng: Shotgun, SMG (tái dùng GunSystem)
Score system: kill point, headshot bonus, combo bonus
Wrist HUD: nhìn vào cổ tay trái thấy ammo + HP
Wave counter và score panel 3D nổi trên tường ảo
Menu start/restart bằng ray pointer
Lưu high score localStorage

File tạo ra: game/WaveManager.js, game/ScoreSystem.js, ui/WristHUD.js, ui/RayPointer.js, weapons/Shotgun.js, weapons/SMG.js
Test: Bấm Start → wave 1: 3 enemy → tiêu diệt hết → bảng điểm hiện → wave 2: 5 enemy nhanh hơn → chơi đến wave 5 → chết → màn hình game over có high score → bấm restart → vào lại từ wave 1 → pass.

Giai đoạn 11 — Depth Occlusion & Light Estimation
Mục tiêu: MR trông thật hơn — vật thật che vật ảo, ánh sáng đồng bộ.
Việc cần làm:

Depth Sensing API: lấy depth texture từ Quest 3 camera mỗi frame
Custom occlusion shader: so sánh depth thật vs depth ảo, discard pixel nếu bị che
Tay thật che được enemy ảo khi giơ ra trước
Light Estimation API: đọc màu, hướng, cường độ ánh sáng phòng
Cập nhật Three.js DirectionalLight + AmbientLight theo thời gian thực
Enemy nhận bóng đổ khớp với ánh sáng phòng thật

File tạo ra: core/DepthSensor.js, core/LightEstimator.js, shaders/occlusion.vert, shaders/occlusion.frag
Test: Giơ tay ra trước enemy → tay thật che một phần enemy ảo → bật đèn phòng → enemy sáng lên khớp → tắt đèn → enemy tối lại → pass.

Giai đoạn 12 — Performance & Polish
Mục tiêu: Chạy ổn định 90fps, sẵn sàng cho người khác chơi.
Việc cần làm:

Instanced mesh cho enemy, đạn, particle
LOD: enemy xa dùng mesh ít polygon
Frustum culling kiểm tra kỹ
Performance monitor: nếu FPS < 72 → tự giảm shadow, particle count
Hologram shader cho UI panel
Toàn bộ gesture có visual feedback (glow tay khi detect)
Comfort options menu: snap turn speed, vignette on/off, dominant hand
Test thật với 10 enemy cùng lúc, nhiều đạn, ragdoll
Build production, deploy Vercel, share link

File tạo ra: core/PerformanceManager.js, shaders/hologram.frag, cập nhật toàn bộ file cũ thêm instancing/LOD
Test: 10 enemy + 30 đạn bay + 3 ragdoll cùng lúc → FPS counter giữ ≥ 72fps liên tục → pass.
